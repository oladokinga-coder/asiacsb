# Настройка Google Таблицы для банка

Чтобы вы видели всех клиентов в одной таблице и могли вручную вносить зачисления/списания (которые сразу отобразятся у клиента в личном кабинете), нужно создать Google Таблицу и подключить её к проекту.

## Шаг 1: Создайте таблицу в Google Sheets

1. Откройте [Google Таблицы](https://sheets.google.com) и создайте новую таблицу.
2. Переименуйте первый лист в **Клиенты**.
3. В первой строке листа «Клиенты» введите **точно** такие заголовки (например, A1–K1):

   | userId | email | fullName | balance | cardNumber | cardValid | cardCvv | passportSeries | passportNumber | birthDate | country | createdAt | beneficiary |
   |--------|-------|----------|---------|------------|-----------|---------|----------------|-----------------|-----------|---------|-----------|-------------|

   - **passportSeries**, **passportNumber**, **birthDate**, **country** — при регистрации заполняются автоматически (country — код страны, например RU, US).
   - **cardNumber**, **cardValid**, **cardCvv** — после регистрации пусты, вносятся вручную.
   - **beneficiary** — необязательный столбец: если вы внесёте данные, они отобразятся в профиле клиента в разделе «Данные вашего аккаунта».

4. Добавьте второй лист: внизу нажмите «+» и назовите лист **Транзакции**.
5. На листе «Транзакции» в первой строке введите заголовки:

   | id | email | amount | type | description | date | send |
   |----|-------|--------|------|-------------|------|------|

   - **email** — email клиента (как в регистрации). По нему транзакция привязывается к счёту.
   - **amount** — сумма: положительная для зачисления, отрицательная для списания (например, 1000 или -500).
   - **type** — например: «Зачисление», «Списание», «Перевод».
   - **description** — описание операции (видит клиент).
   - **date** — дата в любом формате (например, 08.03.2025 или 2025-03-08).
   - **send** — для отправки письма о зачислении на email укажите **ok**. Письмо отправится один раз при следующем запуске крона.

6. Сохраните таблицу. Скопируйте **ID таблицы** из URL:
   `https://docs.google.com/spreadsheets/d/**ВОТ_ЭТОТ_ID**/edit`

## Шаг 2: Google Cloud и сервисный аккаунт

1. Зайдите в [Google Cloud Console](https://console.cloud.google.com).
2. Создайте проект (или выберите существующий).
3. Включите **Google Sheets API**: «APIs & Services» → «Library» → найдите «Google Sheets API» → Enable.
4. «APIs & Services» → «Credentials» → «Create Credentials» → «Service account».
5. Укажите имя (например, «bank-sheets»), создайте. Скачайте ключ (JSON): нажмите на созданный сервисный аккаунт → вкладка «Keys» → «Add Key» → «Create new key» → JSON. Сохраните файл.
6. Откройте скачанный JSON. Вам нужны:
   - `client_email` — скопируйте в переменную `GOOGLE_SERVICE_ACCOUNT_EMAIL`.
   - `private_key` — скопируйте **целиком** (вместе с кавычками и символами `\n`) в переменную `GOOGLE_PRIVATE_KEY`. В `.env` можно писать в одну строку: `GOOGLE_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----\n...\n-----END PRIVATE KEY-----\n"`.

7. **Важно:** откройте вашу Google Таблицу и нажейте «Настройки доступа». Добавьте пользователя по email из `client_email` (из JSON) и дайте ему права **Редактор**. Иначе приложение не сможет читать/писать таблицу.

## Шаг 3: Переменные окружения

В папке проекта создайте файл `.env` (если его ещё нет) и добавьте:

```
DATABASE_URL="file:./dev.db"
GOOGLE_SHEET_ID=ваш_id_таблицы_из_шага_1
GOOGLE_SERVICE_ACCOUNT_EMAIL=ваш_client_email_из_json
GOOGLE_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----
...
-----END PRIVATE KEY-----"
```

Для `GOOGLE_PRIVATE_KEY` в `.env` можно оставить переносы строк как в исходном ключе; в некоторых средах (например, Vercel) удобнее записать ключ в одну строку с символами `\n` вместо переноса.

## Как вы работаете с таблицей

- **Клиенты**  
  При регистрации в лист «Клиенты» автоматически добавляется строка: userId, email, fullName, **passportSeries**, **passportNumber**, **birthDate** (ГГГГ-ММ-ДД), **country** (код страны из формы), balance=0, пустые cardNumber, cardValid, cardCvv, createdAt.  
  Карта после регистрации пустая — вручную внесите **cardNumber**, **cardValid**, **cardCvv**. Баланс (balance) можно менять вручную.

- **Транзакции**  
  Колонки: **id**, **email**, **amount**, **description**, **date**, **send**.  
  Письмо о зачислении отправляется только по строкам, где в колонке **send** написано **ok**. Остальные поля (id, email, amount, description, date) должны быть заполнены, amount > 0. Письмо уходит на **email** из строки (один раз на строку).  
  Операции появятся у клиента в разделе «История» и на главной кабинета. Баланс в листе «Клиенты» обновляйте отдельно.

- **Проверка новых строк**  
  Каждую минуту вызывается адрес  
  `GET` или `POST` **/api/cron/process-transaction-emails**  
  (настройте крон раз в 1 минуту: cron-job.org или Vercel Cron — `* * * * *`).  
  Если в `.env` задан `CRON_SECRET` или `ADMIN_SECRET`, передавайте его в заголовке:  
  `x-cron-secret: ваш_секрет` или `x-admin-secret: ваш_секрет`.

После сохранения таблицы и перезапуска приложения данные из неё будут подтягиваться в личный кабинет клиента.
